Krótka instrukcja obsługi nowego współlokatora w naszej głowie.

Otwarcie. „Nie odpowiem dziś, czy AI zabierze wam pracę w 2042 roku. Spróbujemy odpowiedzieć na trudniejsze pytanie: które części myślenia warto jej oddać, a których oddać nie wolno?”. Poproś, by publiczność zapamiętała swoją odpowiedź do końca.

Teza na dziś

AI raczej nie zastąpi
naszej głowy.

Ale zastąpi wiele ruchów, które głowa dotąd wykonywała sama.

I właśnie dlatego trzeba nauczyć się prowadzić, zanim włączymy autopilota.
Zaznacz różnicę między narzędziem i odpowiedzialnością. Kalkulator nie „zastąpił matematyki”, ale zmienił, które rachunki wykonujemy ręcznie. AI ma większy zakres, więc zmiana jest poważniejsza.

Krok 2 · Attention

Każdy token pyta:
„Na kogo mam patrzeć?”

Ola podała Mai jej zeszyt, bo ona…

Atencja przypisuje wcześniejszym tokenom różne wagi. Nie rozwiązuje całej zagadki, ale buduje kontekst.

Powiedz: „Wyobraźcie sobie klasową imprezę, na której każdy słucha wszystkich, ale nie każdego równie mocno”. Zapytaj: kim jest „ona”? Język bywa niejednoznaczny. Mechanizm atencji pomaga śledzić zależności, lecz sam nie gwarantuje poprawnej interpretacji.

Dokładny skład danych i wszystkie receptury treningowe współczesnych modeli nie są publiczne.

Rozdziel trzy rzeczy. Pre-trening tworzy ogólne kompetencje. Post-trening kształtuje zachowanie. Agent dodaje pętlę działania i zewnętrzne narzędzia. Nie mów, że model „uczy się z każdej rozmowy” w czasie rzeczywistym. Wspomnij, że procesy nowszych modeli są tylko częściowo opisane publicznie.

Największy skok nie brzmi: „AI zna więcej wzorów”. Brzmi: AI dostała pętlę sprawdzania.

To centralny slajd. Podkreśl różnicę między pamiętaniem wyniku i procedurą dochodzenia do wyniku. Matematyka i programowanie mają stosunkowo tani sygnał poprawności. To nie eliminuje błędów, ale pozwala je wykrywać częściej niż w pytaniach o gust, sens lub wartość pomysłu.

Czy 0,999… naprawdę równa się 1?

Droga algebraiczna

x = 0,999…

10x = 9,999…

9x = 9

x = 1

Droga „bez szczeliny”

Gdyby 1 − 0,999… było dodatnie,

musiałoby być mniejsze od 0,1; 0,01; 0,001; …

Nie ma dodatniej liczby mniejszej od wszystkich.

Model może podać pięć dowodów. Ty wybierasz ten, który buduje twoją intuicję.

Zapytaj o pierwszą reakcję: „to prawda” czy „to oszustwo zapisu?”. Pokaż dwa wyjaśnienia. Druga droga jest celowo mniej formalna dla młodszych. W liceum można dopowiedzieć o granicy ciągu lub szeregu geometrycznym. To przykład wartości AI jako generatora różnych wyjaśnień.
Prawdziwa matematyka · pułapka

Dowód, że 1 = 2. Gdzie jest przestępstwo?

Załóżmy: a = b

a² = ab

a² − b² = ab − b²

(a − b)(a + b) = b(a − b)

a + b = b ← dzielimy przez a − b

2b = b, więc 2 = 1

ALARM a − b = 0

Dzielenie przez zero przebrało się za zwykłe skracanie.

Błąd może mieć świetną składnię, garnitur i bardzo pewny ton.

Nie pokazuj alarmu od razu. Daj publiczności trop: „Który krok wymaga warunku?”. To modelowy przykład halucynacji matematycznej: lokalnie płynne kroki, jeden niedozwolony ruch, globalnie absurdalny wynik. Dobre pytanie do AI: „Wskaż pierwszy niepoprawny krok i nazwij warunek”.

Sam model: „Wydaje mi się, że działa.” Agent: „Uruchomiłem. Nie działało. Poprawiłem. Oto test.”

Oficjalny podręcznik Codex: pętla model → działania → weryfikacja; nacisk na testy i przegląd.

Codex jest agentem programistycznym: może czytać repozytorium, edytować pliki i uruchamiać polecenia. To ważne także w matematyce obliczeniowej: kod staje się laboratorium. Nadal trzeba ocenić, czy test sprawdza właściwą rzecz i czy eksperyment numeryczny nie jest mylony z dowodem.
Mini-demo bez internetu

Kiedy pęka piękny wzór Eulera?

n² + n + 41

Daje liczby pierwsze dla n = 0, 1, 2, … 
for (let n = 0; ; n++) {
  const value = n*n + n + 41;
  if (!isPrime(value)) {
    return { n, value };
  }
}
$ testujemy hipotezę… czekam na uruchomienie

Eksperyment znajduje kontrprzykład. Dowód wyjaśnia, dlaczego musi się pojawić.

Kliknij przycisk. Wynik: n=40, wartość 1681=41². Zapytaj „czy to przypadek?”. Dla n=40 mamy 40²+40+41=1681=41². To przykład współpracy: narzędzie znajduje, człowiek lub model dopowiada strukturę. Eksperyment nie dowodzi, że wcześniejsze wartości są pierwsze, dopóki ich nie sprawdzimy.

Najlepszy test nauki: czy potrafisz odtworzyć ideę bez okna czatu i zastosować ją w nowym zadaniu. Uczniowie mogą prosić AI o rolę sokratejskiego korepetytora, generator kontrprzykładów, recenzenta rozwiązania lub autora krótkiego quizu.

KONTEKSTCo już wiem, mam i czego próbowałem?
OGRANICZENIACzego nie rób? Jaki poziom, czas, format?
GOTOWE, GDY…Jak sprawdzimy, że wynik jest dobry?
Zamiast: „Zrób zadanie 7.” Spróbuj: „Jestem w 8 klasie. Pokaż jedną wskazówkę. Potem sprawdź mój krok i znajdź pierwszy błąd.”
To cztery elementy zalecane również w aktualnych dobrych praktykach Codex: goal, context, constraints, done when. W szkole „gotowe” powinno często znaczyć: „umiem wyjaśnić metodę i rozwiązać wariant sam”, a nie „mam tekst do oddania”.
Pięć zasad higieny umysłowej

Oddawaj pracę. Nie oddawaj kierownicy.

  1. Najpierw własny ślad.Choćby hipoteza, rysunek albo pytanie.
  2. Proś o krytykę.„Znajdź pierwszy błąd”, nie „potwierdź, że mam rację”.
  3. Wymagaj testu.Źródło, rachunek, kontrprzykład, drugi dowód.
  4. Chroń dane.Nie wklejaj cudzych danych, haseł i prywatnych historii.
  5. Umiej odłączyć.Jeśli bez AI nie potrafisz wyjaśnić wyniku, to jeszcze nie jest twoja wiedza.
Zwróć uwagę na prywatność i zasady szkoły. Nie moralizuj o „oszukiwaniu”; pokaż praktyczny koszt: oddając zadanie, oddaje się trening własnego mózgu. AI może trening wzmocnić albo go ominąć.

Trzy zdania, których warto pilnować

„Czy model rozumie?”

Zależy od definicji rozumienia. Zachowuje się kompetentnie w wielu zadaniach, ale nie mamy prostego testu na ludzkie doświadczenie wewnętrzne.

„Czy AI jest kreatywna?”

Generuje nowe kombinacje i czasem wartościowe rozwiązania. Spór dotyczy nowości, intencji, oceny i autorstwa.

„Czy można jej ufać?”

Tak jak narzędziu o znanym błędzie: tym bardziej, im lepszy mamy test. Nigdy wyłącznie dlatego, że brzmi pewnie.
