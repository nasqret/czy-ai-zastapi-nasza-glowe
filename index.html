Agent + narzędzia

Agent czyta pliki, uruchamia kod, sprawdza wynik i poprawia błędy.

To tutaj Codex przestaje być tylko „oknem czatu”.

Dokładny skład danych i wszystkie receptury treningowe współczesnych modeli nie są publiczne.

Rozdziel trzy rzeczy. Trening wstępny, nazywany też pretreningiem, buduje ogólne umiejętności. Dostrajanie i posttrening kształtują zachowanie. Agent dodaje pętlę działania i zewnętrzne narzędzia. Nie mów, że model „uczy się z każdej rozmowy” w czasie rzeczywistym. Wspomnij, że procesy nowszych modeli są tylko częściowo opisane publicznie.

To centralny slajd. Podkreśl różnicę między pamiętaniem wyniku i procedurą dochodzenia do wyniku. Matematyka i programowanie często dają szybką i jednoznaczną informację zwrotną. To nie eliminuje błędów, ale pozwala je wykrywać częściej niż w pytaniach o gust, sens lub wartość pomysłu.

Czy 0,999… naprawdę równa się 1?

Droga algebraiczna

x = 0,999…

10x = 9,999…

9x = 9

x = 1

Dowód przez różnicę

Gdyby różnica 1 − 0,999… była dodatnia,

musiałaby być mniejsza od 0,1; 0,01; 0,001; …

Taka dodatnia różnica nie może istnieć.

Model może podać pięć dowodów. Ty wybierasz ten, który naprawdę pomaga ci zrozumieć.

Zapytaj o pierwszą reakcję: „to prawda” czy „to oszustwo zapisu?”. Pokaż dwa wyjaśnienia. Druga droga jest celowo mniej formalna dla młodszych. W liceum można dopowiedzieć o granicy ciągu lub szeregu geometrycznym. To przykład, jak AI może podawać różne wyjaśnienia tego samego zjawiska.
Prawdziwa matematyka · pułapka

Dowód, że 1 = 2. Gdzie ukrył się błąd?

Załóżmy: a = b

a² = ab

a² − b² = ab − b²

(a − b)(a + b) = b(a − b)

a + b = b ← dzielimy przez a − b

2b = b, więc 2 = 1

ALARM a − b = 0

Dzielenie przez zero udaje tu zwykłe skracanie.

Codex nie powinien oceniać tylko wyniku. Powinien sprawdzić każdy krok i jego warunki.

Nie pokazuj alarmu od razu. Daj publiczności trop: „Który krok wymaga warunku?”. To modelowy przykład halucynacji matematycznej: lokalnie płynne kroki, jeden niedozwolony ruch, globalnie absurdalny wynik. Potem pokaż ścieżkę Codex. Dobre polecenie nie brzmi „czy ten dowód jest poprawny?”, lecz: „Sprawdź każdy krok, wskaż pierwszy niepoprawny i nazwij złamany warunek”. Codex powinien odczytać założenie a=b, wywnioskować a−b=0 i dopiero wtedy odrzucić skracanie.

Krótki pokaz bez internetu

Kiedy pęka piękny wzór Eulera?

n² + n + 41

Dla n = 0, 1, …, 39 daje liczby pierwsze. 
for (let n = 0; ; n++) {
  const value = n*n + n + 41;
  if (!isPrime(value)) {
    return { n, value };
  }
}
$ testujemy hipotezę… czekam na uruchomienie

Eksperyment znajduje kontrprzykład. Matematyka tłumaczy, skąd się wziął.

Kliknij przycisk. Wynik: n=40, wartość 1681=41². Zapytaj „czy to przypadek?”. Dla n=40 mamy 40²+40+41=1681=41². To przykład współpracy: narzędzie znajduje kontrprzykład, a człowiek lub model dostrzega stojącą za nim zależność. Eksperyment nie dowodzi, że wcześniejsze wartości są pierwsze, dopóki ich nie sprawdzimy.

Pięć zasad higieny umysłowej

Zlecaj pracę. Nie oddawaj kierownicy.

  1. Najpierw spróbuj sam.Zapisz choćby hipotezę, rysunek albo pytanie.
  2. Proś o krytykę.„Znajdź pierwszy błąd”, nie „potwierdź, że mam rację”.
  3. Proś o sprawdzenie.Poproś o źródło, rachunek, kontrprzykład albo drugi dowód.
  4. Chroń dane.Nie wklejaj cudzych danych, haseł ani prywatnych rozmów.
  5. Umiej wyłączyć AI.Jeśli bez niej nie umiesz wyjaśnić wyniku, to jeszcze nie jest twoja wiedza.
Zwróć uwagę na prywatność i zasady szkoły. Nie moralizuj o „oszukiwaniu”; pokaż praktyczny koszt: kiedy oddajesz gotowe zadanie, omijasz trening własnego mózgu. AI może ten trening wzmocnić albo pomóc go ominąć.

Trzy zdania, których warto pilnować

„Czy model rozumie?”

Zależy od tego, co nazywamy rozumieniem. Model radzi sobie z wieloma zadaniami, ale nie mamy prostego testu, który powiedziałby, czy doświadcza świata tak jak człowiek.

„Czy AI jest kreatywna?”

Generuje nowe kombinacje i czasem wartościowe rozwiązania. Spór dotyczy nowości, intencji, oceny i autorstwa.

„Czy można jej ufać?”

Tak jak każdemu narzędziu, które bywa zawodne: tym bardziej, im lepiej potrafimy sprawdzić wynik. Nigdy tylko dlatego, że odpowiedź brzmi pewnie.
